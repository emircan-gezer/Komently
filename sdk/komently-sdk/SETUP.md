# Komently SDK Setup Guide

## Building the SDK

Before using the SDK in other projects, you need to build it:

```bash
cd komently-sdk
npm install
npm run build
```

This will create a `dist/` folder with the compiled JavaScript and TypeScript definitions.

## Using in Local Projects

### Option 1: File Path (Recommended for development)

In your project's `package.json`:

```json
{
  "dependencies": {
    "@komently/sdk": "file:../project-komently/komently-sdk"
  }
}
```

Then run:
```bash
npm install
```

**Note:** Make sure to rebuild the SDK (`npm run build`) after making changes.

### Option 2: npm link (For active development)

1. In the SDK directory:
```bash
cd komently-sdk
npm link
```

2. In your project:
```bash
cd your-project
npm link @komently/sdk
```

**Pros:**
- Changes are reflected immediately (after rebuild)
- Good for active development

**Cons:**
- Can sometimes cause dependency conflicts
- Need to remember to rebuild

### Option 3: Watch Mode (Best for active development)

Run the SDK in watch mode while developing:

```bash
cd komently-sdk
npm run dev
```

This will automatically rebuild when you make changes. Then use `npm link` in your project.

## Verifying Installation

After installation, verify it works:

```typescript
import { createClient } from '@komently/sdk';

const client = createClient({
  baseUrl: 'http://localhost:3000',
  apiKey: 'your-api-key'
});

console.log('SDK loaded successfully!');
```

## Troubleshooting

### Module not found
- Make sure you've built the SDK (`npm run build`)
- Check that `dist/` folder exists
- Verify the path in `package.json` is correct

### Type errors
- Ensure `dist/index.d.ts` exists (generated by build)
- Try deleting `node_modules` and reinstalling

### Changes not reflected
- Run `npm run build` in the SDK directory
- Or use `npm run dev` for watch mode
- If using npm link, make sure you've rebuilt
